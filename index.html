✦ Learning Journey

Week 1: This week, I set up my GitHub Pages portfolio website and created the main structure, including the About, Projects, Skills, and Contact sections. I learned how GitHub repositories connect to GitHub Pages and how to publish a website online. In addition, I started to understand the basic structure of a website using HTML. I learned how different sections are organised using tags such as which helped me build a clearer and more logical layout. I also explored how CSS affects the visual design of a webpage, even though I am still at an early stage of learning it.

Week 2: I am most proud of successfully publishing my website for the first time. It felt rewarding to see my own work online and accessible through a real URL. This experience helped me realise how development tools and platforms work together in practice. The most challenging part was understanding how repository names affect the website URL and how GitHub Pages links to the correct branch. At the beginning, I was confused about why my website was not updating, but after checking my repository settings and file structure, I gradually solved the problem.

Week 3: Next week, I plan to improve the layout and make the design more user-friendly. I would like to add clearer navigation, improve the visual structure, and possibly include a small maths-related feature to demonstrate my programming skills. I also want to continue learning CSS to make my website more visually appealing. Overall, this week has been a very meaningful step in my learning journey. I feel more confident in using GitHub and starting web development, and I am motivated to keep improving my skills.

Week 4:I improved my website by adding a contact form instead of displaying my email address. Users can now enter their name, email, and message, which improves interaction and usability. The form includes required fields to prevent errors and provides feedback when submitted. This change improves several usability heuristics, including error prevention, recognition rather than recall, and visibility of system status.

Week 4:I customised the colour scheme of my website to a soft purple theme. However, I noticed that the text in the header became difficult to read. To improve usability, I adjusted the text colour to white and added a shadow effect to increase contrast. This improves readability and enhances the overall user experience.
